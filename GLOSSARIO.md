# Glossário de Termos - PDPW PoC

- PoC (Prova de Conceito): Implementação curta para validar viabilidade técnica/funcional.
- Backend: Camada de servidor (API, regras de negócio, acesso a dados).
- Frontend: Camada de interface do usuário (React, navegação, formulários).
- API REST: Interface HTTP padronizada para operações (GET, POST, PUT, DELETE).
- Arquitetura Limpa (Clean Architecture): Separação em camadas independentes (Domain, Application, Infrastructure, API) focando regras de negócio.
- DTO (Data Transfer Object): Objeto para entrada/saída de dados na API (evita expor entidades diretamente).
- Repositório (Repository Pattern): Abstração para acesso a dados que isola infraestrutura da lógica de negócio.
- Entity Framework Core (EF Core): ORM do .NET para mapear entidades para o banco de dados.
- Migration (Migração): Script/versionamento de alterações de esquema de banco gerado pelo EF Core.
- Seed (Semeadura): Inserção de dados iniciais no banco (ex.: usuários de teste).
- CRUD: Operações de Criar, Ler, Atualizar e Remover.
- Swagger / OpenAPI: Documentação interativa da API (via Swashbuckle).
- CORS (Cross-Origin Resource Sharing): Política de segurança para permitir chamadas entre origens diferentes.
- Contêiner (Container): Empacotamento da aplicação e dependências para execução isolada (Docker).
- Docker Compose: Orquestra múltiplos contêineres (ex.: API + Frontend + Banco).
- Contêiner Windows / Linux: Sistema base do contêiner (compatibilidade e limitações diferem).
- IaC (Infraestrutura como Código): Definição de infraestrutura via arquivos (Dockerfile, docker-compose.yml).
- TFS / TFVC: Sistema de controle de versão da Microsoft (legado no cliente).
- GitHub: Plataforma de versionamento e colaboração (entrega final da PoC).
- Backup (Dump) de Banco: Cópia/arquivo para restaurar o banco (SQL Server .bak/.bacpac).
- E2E (Fim a Fim): Teste que percorre o fluxo completo do usuário/sistema.
- Teste de Fumaça (Smoke Test): Verificação rápida de funcionamento básico.
- Mock: Simulação de dependência externa para testes/desenvolvimento.
- Pipeline CI/CD: Automação de integração contínua/entrega contínua.
- Runtime .NET: Ambiente de execução do .NET instalado na máquina.
- Release / Debug: Configurações de build para produção/desenvolvimento.
- Nginx: Servidor web reverso usado para servir frontend e proxy de API.
- SQL Server: Banco de dados relacional da Microsoft usado pelo PDPW.
- Vertical Slice: Entregar um fluxo completo (UI + API + dados) ponta a ponta.
- Horizontal Slice: Entregar camadas parciais (ex.: apenas backend) de várias features.
- ADR (Architecture Decision Record): Registro das decisões arquiteturais e seus motivos.
